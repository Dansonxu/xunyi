# 教师端功能逻辑说明

基于当前教师端原型界面与交互梳理，以下内容从专业产品经理视角输出教师端的详细功能逻辑说明。文档覆盖原型中已经展示的页面元素、操作路径、状态规则、数据联动和边界约束，可作为后续 PRD 细化基础稿。

## 一、文档结论

**当前教师端原型的核心定位：以"教师执行教学任务"为中心，围绕课程履约、班级学情数据钻取、学员跟进和个人资料维护形成一个轻量工作台。**

从原型可判断，教师端是一个"授课执行端"：

1. 教师的第一工作入口是课程表，而不是数据报表。
2. 产品把教师高频动作压缩到最短链路，尤其是"看课-点课-开始上课/进入直播间/查看回放"。
3. 班级学情承担辅助决策作用，帮助教师从班级层面逐层下钻到学生个体，做课后跟进和风险识别。
4. 教师端以查看、操作、反馈为主，不承载排课、调课、基础数据维护等强运营动作。

## 二、产品定位与用户场景

### 2.1 产品定位

教师端面向围棋培训机构教师使用，核心价值是帮助教师在一个统一界面内完成以下工作：

1. 快速登录并进入当天教学状态。
2. 查看个人授课安排并识别当前课程状态。
3. 进入课前、课中、课后的关键教学动作。
4. 查看班级整体学习数据和风险预警。
5. 从班级下钻到学生个体，查看整体表现和每节课数据。
6. 维护个人资料、密码和个人教学统计。

### 2.2 典型使用场景

1. **开课前 10 分钟**
   - 教师打开教师端，查看今天/本周课程，点击即将开始的课程，预览课件并进入上课准备状态。

2. **上课进行中**
   - 教师从课程表中快速识别"进行中"课程，直接进入直播间或授课控制台。

3. **课后复盘**
   - 教师查看已结束课程的回放入口，结合课堂表现和学员数据进行复盘。
   - 复盘老师（刘老师）进入复盘视图，查看小班分组课程的回放和学员每节课表现。

4. **课后跟进学员**
   - 教师进入班级学情，点击班级查看学员列表，再点击学员查看每节课数据表现，判断是否需要联系家长。

5. **阶段性教学观察**
   - 教师在班级学情概览页查看班级平均指标，点击班级进入详情查看学员排名和趋势。

## 三、功能架构总览

### 3.1 一级功能模块

当前原型实际呈现以下一级模块：

1. 登录
2. 课程表
3. 班级学情
4. 个人中心

### 3.2 全局公共能力

1. 顶部品牌区与版本识别
2. 顶部消息提醒及未读数展示
3. 当前登录教师身份展示（支持切换老师角色）
4. 左侧导航切换
5. 课程详情弹窗
6. 学员档案弹窗
7. 面包屑导航与返回按钮
8. 页面内筛选与视图切换
9. 退出登录

### 3.3 模块与业务价值对应表

| 模块 | 核心入口 | 主要价值 | 教师高频动作 |
| --- | --- | --- | --- |
| 登录 | 登录页 | 快速进入工作台 | 登录、切换登录方式 |
| 课程表 | 默认首页 | 承接教师最核心授课任务 | 看课、点课、开课、进直播、看回放 |
| 班级学情 | 左侧导航 | 班级→学员→课节逐层数据钻取 | 看班级指标、看学员列表、看每节课数据 |
| 个人中心 | 左侧导航 | 维护个人信息与账号安全 | 改资料、改密码、看教学统计 |

## 四、全局框架与通用交互

### 4.1 顶部框架

顶部区域包含品牌标识、系统名称、版本号、消息提醒、教师身份和退出入口。

1. **品牌区**：建立教师对当前系统的身份认知。
2. **版本号**：承接灰度发布、问题排查和版本感知。
3. **消息提醒**：承接课程提醒、系统通知、调课通知、待办消息等。
4. **教师身份展示**：显示当前教师姓名（如"李老师"），**支持点击切换老师角色**（李老师 ↔ 刘老师），切换后课程表和班级学情同步联动。
5. **退出入口**：主动结束会话，保证账号安全。

### 4.2 教师角色切换逻辑

教师角色切换是全局联动的核心机制：

| 角色 | 授课类型 | 班级规模 | 课程表视图 | 班级学情班级列表 |
| --- | --- | --- | --- | --- |
| 李老师 | 大班课 | 200人/班 | 普通视图（单课程/格） | 4个大班（启蒙A班、入门C班、进阶D班、冲段精英班） |
| 刘老师 | 小班复盘课 | 8人/组 | 复盘视图（双课程/格） | 3个复盘小组（启蒙A班-复盘1组/2组、入门C班-复盘1组） |

切换规则：
1. 点击顶部教师名称区域触发切换。
2. 切换后课程表在"普通视图"和"复盘视图"之间切换。
3. 切换后班级学情的班级列表同步更新为对应角色所带班级。
4. 切换后课程详情弹窗中的学员数同步变化（大班 200人 / 小班 8人）。

### 4.3 左侧导航

左侧导航用于在四个业务模块间切换：

1. 课程表
2. 班级学情
3. 个人中心

交互逻辑：
1. 当前模块高亮显示。
2. 点击其他模块后，仅切换右侧内容区，不刷新整个应用框架。
3. 导航切换应保留当前登录态。

### 4.4 面包屑导航

班级学情内部使用面包屑导航实现层级定位：

1. **班级概览页**：无面包屑，直接显示页面标题"班级学情"。
2. **班级详情页**：`班级学情 > 启蒙A班`，支持点击"班级学情"返回概览。
3. **学生详情页**：`班级学情 > 启蒙A班 > 张小明`，支持点击上级节点返回对应层级。

### 4.5 通用弹窗逻辑

当前原型包含课程详情弹窗和学员档案弹窗。

通用规则：
1. 点击触发源打开弹窗。
2. 支持点击右上角关闭按钮关闭。
3. 支持点击遮罩关闭。
4. 弹窗打开后，背景页禁止误触滚动。
5. 关闭后保留原页面上下文位置。

## 五、登录

### 5.1 功能目标

支持教师以熟悉且低门槛的方式进入系统，尽快进入授课状态。

### 5.2 页面组成

登录页包含以下元素：

1. 系统 Logo 与产品名称
2. 版本号展示
3. 账号密码登录 Tab
4. 手机号验证码登录 Tab
5. 账号输入框
6. 密码输入框
7. 手机号输入框
8. 验证码输入框
9. 获取验证码按钮
10. 记住密码复选框
11. 忘记密码入口
12. 登录按钮

### 5.3 核心流程

1. 教师选择登录方式（账号密码 / 手机号）。
2. 输入对应凭证后点击登录。
3. 校验通过后进入教师端主应用，默认落在课程表页面。

### 5.4 功能逻辑说明

#### 5.4.1 账号密码登录

1. 教师输入账号和密码后点击登录。
2. 密码输入框默认密文显示。
3. 若勾选"记住密码"，则在本机下次打开时自动回填。
4. 登录成功后进入教师端主应用，默认落在课程表页面。

#### 5.4.2 手机号验证码登录

1. 教师输入手机号后点击"获取验证码"。
2. 系统校验手机号格式正确后发送验证码。
3. 验证码输入正确后可完成登录。
4. 适用于弱密码场景、临时登录场景或密码遗忘后的快捷登录场景。

#### 5.4.3 登录方式切换

1. 支持在账号密码登录和手机号登录之间切换。
2. 切换 Tab 时联动显示/隐藏对应的输入表单。

## 六、课程表

### 6.1 功能目标

课程表是教师端的默认首页，也是教师最核心的工作入口。其目标是帮助教师按时间维度快速理解个人授课安排，并在课前、课中、课后进入对应动作。

### 6.2 页面组成

当前原型课程表页面包含：

1. 本周视图切换（本月视图已去掉）
2. 上一周 / 下一周切换
3. 当前日期区间展示
4. 课程状态图例（未开始/进行中/已结束）
5. 以周为列、时间为行的课程时间格
6. 课程卡片（显示课节编号如 L1-S1 和课程名）
7. 左侧时间列（显示具体时间段，如 9:00-10:25）
8. 课程详情弹窗

### 6.3 双视图机制

课程表支持两种视图，通过顶部教师角色切换触发：

#### 6.3.1 普通视图（李老师 - 大班课）

1. 每个时段最多一个课程块。
2. 课程块显示课节编号（如 L1-S1）和课程名称。
3. 学员规模：200人/班。

#### 6.3.2 复盘视图（刘老师 - 小班复盘课）

1. 每个时段可显示两个课程块（同格双课程）。
2. 两个课程块可能显示相同或不同的课节名称。
3. 学员规模：8人/组。
4. 用于复盘老师查看分组后的课程安排。

### 6.4 时间格展示逻辑

#### 6.4.1 时间轴结构

1. 横向为星期维度（周一至周日），纵向为节次维度。
2. 左侧时间列显示具体时间段（如 9:00-10:25），与右侧课程格时间对齐。
3. 无课程时段标记为灰色不可用（slot-unavailable）。

#### 6.4.2 课程块展示

每个课程块展示两项信息：
1. 课节编号（黄色标签，如 L1-S1）
2. 课程名称（如"第1课 认识围棋"）

### 6.5 课程状态逻辑

原型通过颜色直接区分三类课程状态：

1. **未开始**：蓝色（#3b82f6）
2. **进行中**：红色（#e74c3c）
3. **已结束**：灰色（#d9d9d9）

#### 6.5.1 状态判定规则

| 状态 | 判定条件 | 教师主要动作 |
| --- | --- | --- |
| 未开始 | 当前时间 < 课程开始时间 | 查看详情、开始上课、备课 |
| 进行中 | 开始时间 <= 当前时间 <= 结束时间 | 进入直播间 |
| 已结束 | 当前时间 > 课程结束时间 | 查看回放 |

#### 6.5.2 模拟状态演示

原型内置模拟当前时间机制（MOCK_CURRENT_COURSE_TIME），用于演示不同时间点的课程颜色变化：
- 早于模拟时间的课程标记为"已结束"（灰色）
- 等于模拟时间的课程标记为"进行中"（红色）
- 晚于模拟时间的课程标记为"未开始"（蓝色）

### 6.6 课程详情弹窗

#### 6.6.1 展示字段（动态根据状态渲染）

所有状态均显示：班级、课程、时间、学员数。

#### 6.6.2 不同状态下的弹窗差异

| 状态 | 显示字段 | 动作按钮 |
| --- | --- | --- |
| 未开始 | 班级、课程、时间、学员数 | 开始上课 + 备课 |
| 进行中 | 班级、课程、时间、学员数 | 进入直播间（唯一按钮） |
| 已结束 | 班级、课程、时间、学员数 | 查看回放（唯一按钮） |

#### 6.6.3 复盘视图下的 Tab 机制

当在复盘视图中点击课程块时：
1. 弹窗顶部以 Tab 形式显示该时段的两个课程信息。
2. 默认选中点击的课程块对应的 Tab。
3. Tab 切换时联动显示对应课程的详情内容。
4. 按钮区域使用当前选中课程的状态来决定显示哪些按钮。

#### 6.6.4 学员数联动规则

- 普通视图（李老师大班课）：学员数显示 **200人**
- 复盘视图（刘老师小班课）：学员数显示 **8人**

### 6.7 不同课程状态下的动作

#### 6.7.1 未开始课程

1. "开始上课"：高亮显示，点击后跳转直播授课页。
2. "备课"：点击后可进入备课状态（原型中为示意按钮）。

#### 6.7.2 进行中课程

1. "进入直播间"：用于已开课后的继续授课或断线重进。
2. 唯一按钮，占据整行。

#### 6.7.3 已结束课程

1. "查看回放"：用于教师课后复盘。
2. 唯一按钮，占据整行。

## 七、班级学情

### 7.1 功能目标

帮助教师从"班级整体视角"理解教学效果，并支持逐层下钻到"学员个体视角"和"每节课数据视角"，形成完整的数据观察链路。

### 7.2 三层页面结构

班级学情通过三层页面实现数据钻取：

```
班级学情（概览） → 班级详情 → 学生详情
```

#### 7.2.1 第一层：班级概览

页面元素：
1. 页面标题：班级学情
2. 全部班级平均统计卡片（4个）：平均出勤率、平均预习完成率、平均答题正确率、平均课后完成率
3. 班级列表表格：班级名称、学员数、出勤率、预习完成率、答题正确率、课后完成率、操作（查看详情）
4. 每行可点击，点击进入班级详情页

班级列表与老师联动：
- **李老师视图**：显示 4 个大班（启蒙A班、入门C班、进阶D班、冲段精英班），每班 200 人
- **刘老师视图**：显示 3 个复盘小组（启蒙A班-复盘1组/2组、入门C班-复盘1组），每组 8 人

#### 7.2.2 第二层：班级详情

页面元素：
1. 面包屑导航：`班级学情 > 启蒙A班`
2. 返回按钮：返回班级概览
3. 页面标题：班级名称
4. 班级统计卡片（4个）：该班平均出勤率、平均预习完成率、平均答题正确率、平均课后完成率
5. 学员列表表格：姓名、年龄、等级、出勤率、预习完成率、作业完成率、答题正确率
6. 搜索框：搜索学员姓名
7. 每行可点击，点击进入学生详情页
8. 图表区域：班级成绩趋势折线图、学员排名柱状图（placeholder）

#### 7.2.3 第三层：学生详情

页面元素：
1. 面包屑导航：`班级学情 > 启蒙A班 > 张小明`
2. 返回按钮：返回班级详情
3. 页面标题：学生姓名
4. 学生基本信息卡片：头像、姓名、年龄、班级、校区、等级标签、学习状态标签
5. 整体数据表现统计卡片（4个）：出勤率、预习完成率、作业完成率、答题正确率
6. 每节课数据表格：课节、课程名、出勤状态、预习完成、课堂答题正确率、课后作业完成
7. 示例数据约 8 行，覆盖已上过的课程

### 7.3 核心指标说明

| 指标 | 含义 | 业务价值 |
| --- | --- | --- |
| 出勤率 | 学员到课情况 | 判断学习连续性 |
| 预习完成率 | 课前准备情况 | 判断学习主动性 |
| 答题正确率 | 课堂答题表现 | 判断知识吸收效果 |
| 课后完成率 | 作业/课后任务落实情况 | 判断学习闭环情况 |

### 7.4 每节课数据字段

学生详情页的每节课数据表格包含以下字段：

| 字段 | 说明 |
| --- | --- |
| 课节 | 课节编号，如 L1-S1 |
| 课程名 | 课程名称，如"第1课 认识围棋" |
| 出勤状态 | 出勤/缺勤，用颜色标签区分 |
| 预习完成 | 已完成/未完成/待完成，用颜色标签区分 |
| 课堂答题正确率 | 百分比数值 |
| 课后作业完成 | 已完成/未完成/待完成，用颜色标签区分 |

### 7.5 状态标签颜色规则

| 标签类型 | 颜色 | 适用场景 |
| --- | --- | --- |
| 出勤/已完成 | 绿色（tag-green） | 正常完成状态 |
| 待完成/未完成 | 橙色（tag-orange） | 待处理或未完成 |
| 缺勤 | 红色（tag-red） | 异常状态 |
| — | 灰色（tag-gray） | 缺勤时无数据 |

## 八、个人中心

### 8.1 功能目标

支持教师维护个人资料和账号安全信息，同时查看个人阶段性教学产出。

### 8.2 页面组成

当前原型个人中心包含三部分：

1. 个人资料
2. 修改密码
3. 教学统计

### 8.3 个人资料

可见字段：
1. 头像（支持更换）
2. 姓名
3. 围棋段位
4. 手机号（脱敏展示）
5. 个人简介
6. 保存修改按钮

### 8.4 修改密码

页面字段：
1. 当前密码
2. 新密码
3. 确认新密码
4. 确认修改按钮

### 8.5 教学统计

当前原型展示：
1. 本月课时数
2. 学员评价
3. 月度课时趋势图（placeholder）

## 十一、当前原型已体现的重点与不足

### 11.1 已体现的重点

1. **课程表是教师主工作入口**，产品方向正确。
2. **教师角色切换实现全局联动**，课程表和班级学情数据同步变化。
3. **班级学情三层钻取结构清晰**：班级概览 → 班级详情 → 学生详情 → 每节课数据。
4. **课程详情弹窗根据状态分流动作**：未开始/进行中/已结束三种状态显示不同的字段和按钮。
5. **复盘视图支持同格双课程**，并以 Tab 形式在弹窗中展示。

### 11.2 当前原型尚未展开但建议后续补齐的内容

1. 登录校验与账号安全策略。
2. 消息中心详情页和消息分类。
3. 课程取消、调课、冲突等异常状态展示。
4. 图表的实际交互能力与指标明细查看。
5. 个人资料字段的编辑权限和审核机制。
6. 学情数据的实时刷新机制。
7. 学员档案从班级学情学生详情页的快捷入口。

## 十二、结语

**从当前原型判断，教师端已经具备较清晰的核心骨架，尤其是"教师角色切换 + 课程表双视图 + 班级学情三层钻取"三大核心链路已经贯通。下一步最关键的是把状态规则、指标口径、异常流程、图表交互补完整。**

如果后续要进入正式 PRD 阶段，建议优先细化以下三块：

1. 课程状态与授课动作闭环。
2. 教师角色切换后的数据权限和班级归属规则。
3. 学情指标口径与每节课数据的采集来源。
