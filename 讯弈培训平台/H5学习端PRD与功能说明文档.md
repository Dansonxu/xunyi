## 第二部分　产品概述与角色定位

### 2.1 产品定位

H5 学习端是面向"代理商"与"内部员工"的移动学习门户，承担三大职责：

1. **发现学习内容**：按分类查找资料、浏览培训项目、查看待考考试；
2. **完成学习与考核**：在线观看/下载资料、自动记录学习进度、参加考试并即时获得成绩；
3. **查看个人成长**：查看学习记录、考试成绩、培训项目完成情况。

所有可学习内容与可参加考试均来自管理后台的配置；学员无法创建或修改内容。

### 2.2 角色与使用场景

| 角色     | 使用场景                                                             |
| -------- | -------------------------------------------------------------------- |
| 代理商   | 学习产品知识、销售政策、售后服务资料，参加对应考核，完成必修培训项目 |
| 内部员工 | 学习入职培训、合规培训、制度资料，参加对应考核，完成必修培训项目     |

### 2.3 可见性规则（核心前置规则）

学员在 H5 端可见某内容，须**同时满足**两个条件：

1. 内容发布状态 = "已发布"；
2. 内容受众 = 学员自身角色，或内容受众 = "全部"。

由此推论：草稿内容不可见；"代理商"学员看不到仅面向"内部员工"的内容，反之亦然。

## 第四部分　页面结构说明

### 4.1 整体框架

H5 采用"顶部导航 + 页面内容 + 底部 Tab 栏"三段式布局，适配移动端单手操作。

```
┌─────────────────────┐
│  顶部标题栏（可选返回/操作） │
├─────────────────────┤
│                     │
│      页面内容        │
│                     │
├─────────────────────┤
│ 首页 │ 资料 │ 培训 │ 考试 │ 我的 │
└─────────────────────┘
```

### 4.2 各区域职责

| 区域        | 职责                                                                |
| ----------- | ------------------------------------------------------------------- |
| 顶部标题栏  | 显示当前页面标题；非一级 Tab 页提供返回按钮；部分页面提供右上角操作 |
| 内容区      | 承载列表、卡片、表单、图表等具体信息                                |
| 底部 Tab 栏 | 五大主入口：首页、资料、培训、考试、我的；当前 Tab 高亮             |

### 4.3 底部 Tab 结构

| Tab 名称 | 图标语义 | 跳转页面               |
| -------- | -------- | ---------------------- |
| 首页     | 房子     | index.html             |
| 资料     | 书本     | materials.html         |
| 培训     | 毕业帽   | training-programs.html |
| 考试     | 试卷     | exams.html             |
| 我的     | 用户     | profile.html           |

### 4.4 信息架构

```
首页（仪表盘）
├── 待学资料数 / 待考考试数 / 最近成绩
├── 公告
├── 最新资料（入口 → 资料中心 / 资料详情）
├── 待考考试（入口 → 考试须知 / 考试中心）
└── 我的培训项目（入口 → 培训项目）

资料
├── 资料中心（类型 + 分类筛选）
└── 资料详情（学习 + 进度记录）

培训
├── 培训项目列表
└── 培训项目详情（资料 + 考试清单）

考试
├── 考试中心
├── 考试须知
├── 答题页
└── 考试结果

我的
├── 个人信息
├── 学习记录
├── 我的成绩
└── 切换演示用户
```

## 第五部分　全局通用规则

以下规则贯穿 H5 各页面，后续模块不再重复说明。

### 5.1 学习者身份规则

- H5 端学习者只能是 `role = agent` 或 `role = employee` 的用户；
- 无登录态时（Demo 模式）自动兜底为第一个代理商账号，保证体验连续性；
- 学习者身份决定所有内容的可见范围。

### 5.2 内容可见性规则

详见 2.3。内容列表页统一通过 `DataStore.getVisibleMaterials(role)` / `getVisibleExams(role)` / `getVisibleTrainingPrograms(role)` 获取。

### 5.3 通用交互反馈

| 场景           | 统一反馈                                                             |
| -------------- | -------------------------------------------------------------------- |
| 操作成功/失败  | 轻提示（Toast），自动消失；成功绿色对勾，失败红色叉号                |
| 无权限访问     | Toast 提示"资料/考试/项目不存在或无权限查看"，并返回上一页或跳转列表 |
| 列表为空       | 显示"暂无"空状态占位（带图标），不留空白                             |
| 交卷等关键操作 | 弹窗二次确认                                                         |

### 5.4 角色标签样式

- 代理商：蓝色标签；
- 内部员工：绿色标签。

## 第六部分　功能详细说明

---

### 6.1 首页

**功能定义**：学员的学习仪表盘，一眼呈现当前学习任务的待办与最新动态。

**前置条件**：H5 端已识别当前学习者身份。

**页面构成与交互逻辑**：

1. **顶部欢迎区**：显示"欢迎，{姓名}"及"继续学习，提升{代理/员工}业务知识"，按当前角色动态变化。
2. **核心统计卡**：横向三卡，分别为：
   - 待学资料数：学员可见且尚未"已完成"的资料数；
   - 待考考试数：学员可见且剩余考试次数 > 0 的考试数；
   - 最近成绩：最近一次考试成绩，无则"-"。
3. **公告条**：喇叭图标 + 最新一条公告标题（取 announcements 首条），仅展示不可点击。
4. **最新资料**：展示学员可见的最新 3 份资料卡片，含类型、分类、进度条；点击跳转资料详情。
5. **待考考试**：展示学员可见且可考的最多 3 场考试卡片，含时长、题量、及格分；点击跳转考试须知。
6. **我的培训项目**：若学员可见的培训项目非空，在公告下方插入展示前 2 个项目，含资料数、考试数、是否必修、完成进度条。

**输出结果/异常处理**：

- 无资料：最新资料区显示"暂无资料"空状态；
- 无待考考试：待考考试区显示"暂无待考考试"空状态；
- 无培训项目：不展示"我的培训项目"区块。

**关联影响**：

- 首页所有数据均来自其他模块的读取，不产生写操作；
- 学习/考试行为会回流到首页统计，刷新后更新。

---

### 6.2 资料中心

**功能定义**：学员查找与浏览学习资料的入口，支持按类型和分类筛选。

**前置条件**：H5 端已识别学习者身份。

**操作流程**：进入页面 → 自动加载学员可见资料 → 可按类型/分类筛选 → 点击资料卡片进入详情。

**筛选器规则**：

| 筛选器 | 选项                      | 交互                                               |
| ------ | ------------------------- | -------------------------------------------------- |
| 类型   | 全部 / PDF / Word / 视频  | 横向标签，点击高亮并即时刷新列表                   |
| 分类   | 全部分类 + 后台配置的分类 | 横向标签，只有存在分类时才展示；点击高亮并即时刷新 |

**列表字段与展示规则**：

| 字段     | 说明                                                       |
| -------- | ---------------------------------------------------------- |
| 资料名称 | 标题                                                       |
| 类型图标 | PDF 用文件 PDF 图标，Word 用文件 Word 图标，视频用播放图标 |
| 分类     | 所属分类                                                   |
| 状态     | "已完成"（绿色）/ "未学习"（黄色）                         |
| 进度条   | 当前学习进度百分比                                         |

**输出结果/异常处理**：筛选无结果 → 显示"暂无符合条件的资料"。

**关联影响**：点击跳转资料详情，进入即开始记录学习行为。

---

### 6.3 资料详情

**功能定义**：展示单份资料的内容并自动记录学习进度。

**前置条件**：学习者已登录；资料存在且对当前角色可见。

**操作流程**：

1. 进入详情页，系统校验可见性；
2. 自动调用 `startStudy()` 标记学习开始（浏览数 +1）；
3. 根据资料类型渲染不同学习方式；
4. 学习过程中持续更新进度；
5. 达到 100% 时标记"已完成"（完成数 +1）。

**不同资料类型的交互规则**：

| 类型 | 学习方式           | 进度计算                                                          |
| ---- | ------------------ | ----------------------------------------------------------------- |
| 视频 | HTML5 播放         | 按播放时间 / 总时长实时计算；播放结束时强制 100%                  |
| PDF  | iframe 内嵌预览    | 按页面停留时长估算：30 秒视为完成；切到后台暂停计时，返回前台续计 |
| Word | 提供"下载查看"按钮 | 点击下载按钮即视为完成，进度直接跳到 100%                         |

**进度规则**：

- 进度取历史最大值，不回退；
- 达到 100% 后状态自动变为"已完成"，并记录完成时间；
- 非 100% 的进度在达到 100% 或每增加 5% 时持久化一次（降低写入频次）。

**输出结果**：进度更新、Toast "学习完成"（达到 100% 时）。

**异常处理**：

- 资料不存在或不可见 → Toast "资料不存在或无权限查看"，1 秒后返回上一页；
- 已完成的资料再次进入显示"已完成"并仍可预览。

**关联影响**：学习进度与学习完成状态回流到首页统计、培训项目进度、学习记录、后台学习统计。

---

### 6.4 培训项目（列表）

**功能定义**：展示学员需要完成的培训计划集合，便于按项目维度推进学习。

**前置条件**：H5 端已识别学习者身份。

**列表字段与展示规则**：

| 字段     | 说明                               |
| -------- | ---------------------------------- |
| 项目名称 | 标题                               |
| 必修标识 | 必修项目带红色"必修"标签           |
| 内容构成 | "N 个资料 / M 场考试"              |
| 完成状态 | "已完成"（绿色）/ "进行中"（黄色） |
| 进度条   | 项目整体完成百分比                 |

**项目完成度计算规则**：

```
进度 =（已完成资料数 + 已通过考试数）/（资料总数 + 考试总数）
全部完成 = 已完成资料数 + 已通过考试数 == 资料总数 + 考试总数
```

**排序规则**：按项目排序号升序。

**输出结果/异常处理**：无项目 → 显示"暂无培训项目"空状态。

**关联影响**：点击项目进入项目详情。

---

### 6.5 培训项目（详情）

**功能定义**：展示单个培训项目的全部内容清单，学员可按清单逐项完成。

**前置条件**：学习者已登录；项目存在且对当前角色可见。

**页面构成**：

1. **项目概览卡**：项目名称、简介、是否必修、整体进度条；
2. **项目资料区**：列出项目包含的全部资料，展示每个资料的学习状态与进度；点击跳转资料详情；
3. **项目考试区**：列出项目包含的全部考试，展示通过状态与剩余次数；点击跳转考试须知；次数为 0 时卡片置灰。

**考试卡片状态规则**：

| 状态           | 展示文案                  |
| -------------- | ------------------------- |
| 已通过         | "已通过"（绿色）          |
| 未通过但有次数 | "剩余 N 次"（黄色）       |
| 次数已用完     | "次数已用完"（灰色/置灰） |

**输出结果/异常处理**：项目不存在或不可见 → 显示"项目不存在或无权限查看"空状态。

**关联影响**：完成的资料/考试会同步更新项目整体进度。

---

### 6.6 考试中心

**功能定义**：列出学员当前可见且可参加的全部考试。

**前置条件**：H5 端已识别学习者身份。

**列表字段与展示规则**：

| 字段     | 说明                                              |
| -------- | ------------------------------------------------- |
| 考试名称 | 标题                                              |
| 考试信息 | 时长、题量、及格分                                |
| 剩余次数 | "剩余 N 次"（可点击）/ "次数已用完"（置灰不可点） |
| 已考次数 | "已考 N 次"                                       |

**点击规则**：

- 剩余次数 > 0：跳转考试须知；
- 剩余次数 = 0：不可点击，卡片透明度降低。

**输出结果/异常处理**：无考试 → 显示"暂无考试"空状态。

**关联影响**：进入考试须知页。

---

### 6.7 考试须知

**功能定义**：考前确认考试信息与规则，并校验是否满足开考条件。

**前置条件**：学习者已登录；考试存在且可见；剩余次数 > 0（由考试中心控制进入，但页面仍需再次校验）。

**页面构成**：

1. **考试信息**：考试名称、时长、题目数量、及格分数、剩余次数；
2. **考试规则**：
   - 请在规定时间内完成答题，超时将自动交卷；
   - 答题过程中请保持页面打开，刷新页面可恢复已答题目；
   - 交卷后可查看成绩与答案解析；
   - 请诚信考试，独立完成。
3. **关联资料完成检查**：若考试关联了学习资料，列出这些资料及完成状态；
4. **开始考试按钮**：根据状态显示不同文案与行为。

**按钮状态规则**：

| 场景                 | 按钮文案           | 行为                                     |
| -------------------- | ------------------ | ---------------------------------------- |
| 存在未完成的关联资料 | "请先完成关联资料" | 点击跳转第一份未完成资料详情             |
| 剩余次数 = 0         | "考试次数已用完"   | 禁用，不可点击                           |
| 满足开考条件         | "开始考试"         | 清除历史答题会话，保存新会话，跳转答题页 |

**输出结果**：满足条件时跳转 exam-answer.html。

**异常处理**：考试不存在或不可见 → Toast 并返回上一页。

---

### 6.8 答题页

**功能定义**：学员正式作答的页面，支持单选、多选、判断三种题型，带倒计时与答题暂存。

**前置条件**：学习者已登录；考试存在且可见；剩余次数 > 0。

**页面构成**：

1. **顶部栏**：倒计时、已答题数 / 总题数；
2. **题目区**：题号、题型标签、题干、选项；
3. **题号导航**：一排题号，已答题高亮，当前题高亮边框，点击可跳转；
4. **底部操作**：上一题、下一题、交卷按钮。

**题型交互规则**：

| 题型   | 选择方式                              |
| ------ | ------------------------------------- |
| 单选题 | 点击选项，单选互斥                    |
| 多选题 | 点击选项，可多选，再次点击取消        |
| 判断题 | 渲染为两个选项（正确/错误），单选互斥 |

**倒计时规则**：

- 从进入页面开始倒计时，显示格式 MM:SS；
- 时间到 0 自动交卷，并提示"考试时间到，自动交卷"；
- 倒计时基于 `startTime` + `duration` 计算。

**答题会话规则**：

- 进入答题页时，优先恢复同一场考试的答题会话（`examId` 匹配）；
- 作答后实时保存答案到 sessionStorage，刷新页面可恢复；
- 提交后清除会话。

**交卷规则**：

- 点击"交卷"时，若存在未答题目，弹窗提示"还有 N 题未答，确定交卷吗？"；
- 确认后提交，计算得分，跳转考试结果页。

**输出结果**：提交成功跳转 exam-result.html?recordId=xxx。

**异常处理**：

- 考试不存在或不可见 → Toast 并返回考试中心；
- 时间到自动交卷。

**业务规则（评分）**：

- 每题：用户答案集合与正确答案集合完全一致才得分（多选不可多选、不可漏选）；
- 总分 = 答对题目分值之和；
- 得分 ≥ 及格分 → "passed"，否则 "failed"。

**关联影响**：考试结果写入考试记录，影响考试中心剩余次数、我的成绩、培训项目进度、后台考试统计。

---

### 6.9 考试结果

**功能定义**：展示本次考试结果与答案解析。

**前置条件**：已完成一场考试并拿到 recordId。

**页面构成**：

1. **得分卡**：
   - 大字显示得分；
   - 通过/未通过标签；
   - 答对题数、考试用时；
2. **答案解析**：逐题展示：
   - 题干；
   - 你的答案（标对/错）；
   - 正确答案。

**解析展示规则**：

- 判断题答案以"正确/错误"展示；
- 选择题答案以"选项编号 + 选项文本"展示；
- 未作答显示"未作答"并判错。

**输出结果/异常处理**：记录不存在 → Toast "考试记录不存在"，跳转考试中心。

**关联影响**：纯展示，不产生写操作。

### 6.10 我的（个人中心）

**功能定义**：展示学员身份信息并提供个人相关入口。

**前置条件**：H5 端已识别学习者身份。

**页面构成**：

1. **个人信息区**：头像（姓名首字）、姓名、角色标签；
2. **功能菜单**：
   - 学习记录（跳转 my-study.html）；
   - 我的成绩（跳转 my-scores.html）；
3. **切换演示用户**：Demo 模式下用于快速切换代理商/员工身份体验。

**输出结果**：点击菜单跳转对应页面；切换用户后 Toast 提示当前身份。

**关联影响**：切换用户会改变当前学习者，从而影响后续所有页面内容可见性。

### 6.11 学习记录

**功能定义**：展示学员全部资料学习历史，按时间倒序。

**前置条件**：H5 端已识别学习者身份。

**列表字段**：资料名称、状态（已完成/学习中）、进度百分比、时间（完成时间优先，否则开始时间）。

**排序规则**：按完成时间/开始时间倒序，最近的学习记录在最上方。

**输出结果/异常处理**：无记录 → 显示"暂无学习记录"空状态。

**关联影响**：点击单条记录跳转资料详情，可继续学习。

### 6.12 我的成绩

**列表字段**：考试名称、通过/未通过、答对/总题数、提交时间、得分。

**排序规则**：按提交时间倒序，最近的考试在最上方。

**输出结果/异常处理**：无记录 → 显示"暂无考试成绩"空状态。

**关联影响**：纯展示，不产生写操作。

## 第八部分　状态与流转

### 8.1 资料学习状态（学员维度）

| 状态   | 含义                        | 流转条件                                |
| ------ | --------------------------- | --------------------------------------- |
| 未学习 | 无学习记录                  | 进入资料详情 → 学习中                  |
| 学习中 | 已产生学习记录，进度 < 100% | 继续学习，进度增加；达到 100% → 已完成 |
| 已完成 | 进度 = 100%                 | 终态，可重复预览                        |

### 8.2 培训项目完成状态

| 状态   | 判定                                          |
| ------ | --------------------------------------------- |
| 进行中 | 项目内（已完成资料数 + 已通过考试数）< 总项数 |
| 已完成 | 项目内（已完成资料数 + 已通过考试数）= 总项数 |

### 8.3 考试结果状态

| 状态   | 判定           |
| ------ | -------------- |
| 通过   | 得分 ≥ 及格分 |
| 未通过 | 得分 < 及格分  |

### 8.4 考试可考状态

| 状态       | 判定                     |
| ---------- | ------------------------ |
| 可考       | 已考次数 < 最大考试次数  |
| 次数已用完 | 已考次数 ≥ 最大考试次数 |

## 第九部分　权限说明

H5 学习端只有一个抽象权限主体：**学习者**。学习者内部仅按角色区分可见内容，无操作权限差异。

| 能力                         | 学习者（代理商） | 学习者（内部员工） |
| ---------------------------- | ---------------- | ------------------ |
| 查看面向自身角色的已发布内容 | ✅               | ✅                 |
| 查看面向对方角色的内容       | ❌               | ❌                 |
| 查看"全部"类型内容           | ✅               | ✅                 |
| 学习资料、参加考试           | ✅               | ✅                 |
| 查看个人学习记录与成绩       | ✅               | ✅                 |
| 创建/修改/删除内容           | ❌               | ❌                 |

所有内容的新增、编辑、删除、发布均由 Web 管理后台完成，H5 端只读消费。
