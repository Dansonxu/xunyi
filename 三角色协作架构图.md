## 二、教务端任务详解（6大域）

### 2.1 师资管理

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 教师招募 | 录入教师档案（姓名/电话/类型/级别） | 运营端 | 教师管理 |
| 可排周次 | 配置教师可上课的星期（星期级） | 运营端 | 教师管理 |
| 教师考勤 | MASTER/TUTOR 双角色考勤，上下播自动签到签退 | 运营端 | 考勤管理（教师考勤Tab） |
| 教师课表 | 教师Web端查看个人课表 | 教师端 | 课程表 |

### 2.2 班级管理

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 创建班级 | 录入班级基础信息（名称/门店/开班日期） | 运营端 | 班级管理 |
| 开班排课向导 | **5步流程**：班级信息 → 选课包 → 分配表 → 课表预览 → 确认开班 | 运营端 | 开班排课向导 |
| 状态流转 | DRAFT → PREPARED → TEACHING → ENDED | 运营端 | 班级管理 |
| 人员调整 | 从PAID订单加入学员、迁移班级、移除学员 | 运营端 | 班级管理 |

### 2.3 学员管理

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 订单入班 | 仅PAID订单可操作入班/换班 | 运营端 | 学员订单管理 |
| 家长学员档案 | 客户档案 + 家庭关系 + 消费轨迹 | 运营端 | 家长学员档案 |
| 插班/转班/停课 | 学籍变动6类型：ENROLL/TRANSFER_CLASS/SUSPEND/RESUME/UPGRADE/LEAVE | 运营端 | 插班/转班/停课 |

### 2.4 排课管理

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 生成课表 | 创建Schedule + Period（85min单层） | 运营端 | 排课管理 |
| 课表查看 | 月/周视图切换，4维度（班级/主讲/复盘/学生） | 运营端 | 排课管理 |
| 调课/重排 | 单节调课，重排限未来未开课节 | 运营端 | 排课管理 |
| 教师备课 | 按直播关卡读课题/课件/课中题/课后题 | 教师端 | 课程表（备课按钮） |

### 2.5 考勤消课

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 学员考勤 | 每学员每节课2条（BIG大班段 + SMALL小班段） | 运营端 | 考勤管理 |
| 教师考勤 | MASTER→BIG段，TUTOR→SMALL段 | 运营端 | 考勤管理 |
| 消课扣减 | **仅BIG段 + 已消课状态才扣1课时** | 运营端 | 考勤管理 |
| 异常预警 | 长期未消课/结业未消完/停课超期 | 运营端 | 消课数据大盘 |

### 2.6 教学反馈

| 子任务 | 功能说明 | 原型文件 | 页面 |
|--------|---------|---------|------|
| 学习报告 | 6维能力雷达图（吃子/逃跑/做眼/对杀/布局/收官） | 运营端+教师端 | 学员档案/班级学情 |
| 消课大盘 | 消课统计、消课率、异常学员 | 运营端 | 消课数据大盘 |
| 课后评价 | 家长1-5星评价 + 文字 | 运营端 | 学员档案 |

## 四、每条关联的详细说明

### 关联①：教研·地图管理 → 销售·课包管理

```
方向：教研 → 销售
数据流：Map (地图) → CoursePackage.mapItems
关联说明：
  • 教研在「地图管理」中创建地图，配置节奏规则（X预习+1直播+Y宝箱）
  • 销售在「课包管理」中引用地图，标总课时数
  • 一张地图可被多个课包引用
  • 课包状态=启用后，可被商品引用上架

原型对应：
  运营端 → 内容管理·地图管理
  运营端 → 商城管理·课包管理
```

### 关联②：销售·课包管理 → 销售·商品管理

```
方向：销售内部
数据流：CoursePackage + PhysicalItem + EntitlementPackage → Goods
关联说明：
  • 商品三合一：M课包 + N实物 + K权益包
  • 底层资源各自独立，商品只引用不内嵌
  • 家长端只认商品SKU，底层资源不直售
  • 底层资源禁用 → 引用商品自动下架

原型对应：
  运营端 → 商城管理·课包管理 / 实物管理 / 权益包管理
  运营端 → 商城管理·商品管理
```

### 关联③：销售·课包管理 → 教务·排课管理

```
方向：销售 → 教务
数据流：CoursePackage → Schedule / Period
关联说明：
  • 教务在开班排课向导Step2选择课包
  • Step3按地图分配课时数（校验总和=课包总课时）
  • Step5确认开班后生成Schedule（每节课对应一个直播关卡）
  • Schedule.mapId / mapStageId / sessionSeq 关联到教研的地图关卡

原型对应：
  运营端 → 教务管理·开班排课向导
  运营端 → 教务管理·排课管理
```

### 关联④：教研·直播关卡 → 教务·教师备课

```
方向：教研 → 教务
数据流：MapStage (直播关卡) → 教师备课内容
关联说明：
  • 教研为每个直播关卡配置：直播课题、课件资源、课中题列表、课后题列表
  • 教师在教师端按课表对应的直播关卡备课
  • 直播课题三端共用同一个名字（家长端/学员App/教师Web）

原型对应：
  运营端 → 内容管理·地图管理（关卡配置）
  教师端 → 课程表（备课按钮）
```

### 关联⑤：教研·能力维度 → 教务·学习报告

```
方向：教研 → 教务
数据流：AbilityDimension → LearningReportSnapshot
关联说明：
  • 教研定义6维能力：吃子、逃跑、做眼、对杀、布局、收官
  • 学员在App学习过程中产生行为数据
  • 系统每天0:30重算学习报告快照（6维雷达图）
  • 三端共享同一API同一字段：学员App / 家长端 / 运营端教务 / 教师端

原型对应：
  运营端 → 内容管理·能力维度
  运营端 → 教务管理·学员档案（学习报告Tab）
  教师端 → 班级学情（待补充6维雷达图）
```

### 关联⑥：销售·订单管理 → 教务·学员管理

```
方向：销售 → 教务
数据流：Order (PAID) → ClassStudent
关联说明：
  • 家长下单支付后，订单状态=PAID
  • 系统触发：学员加入「待开班」班级
  • 仅PAID订单可操作入班/换班
  • 订单归属：家长（付费方）→ 学员（消耗方）→ 门店（归属链）

原型对应：
  运营端 → 商城管理·订单管理
  运营端 → 教务管理·学员订单管理 / 班级管理
```

### 关联⑦：教务·考勤消课 → 销售·分佣结算

```
方向：教务 → 销售
数据流：StudentAttendanceRecord → Settlement / Profit
关联说明：
  • 教务产出消课数据：仅BIG段 + 已消课状态才计入
  • 销售/财务按周期汇总消课，按分佣比计算
  • 推广人业绩单独看板
  • 总部定期对公打款，代理端无提现按钮

原型对应：
  运营端 → 教务管理·考勤管理 / 消课数据大盘
  运营端 → 销售管理·课消分佣 / 结算管理
```

## 七、关键设计原则（影响三角色协作）

| 原则 | 说明 | 影响角色 |
|------|------|---------|
| **商品三层独立** | CoursePackage/PhysicalItem/EntitlementPackage各自独立，Goods只引用不内嵌 | 销售 |
| **家长端只认商品** | 底层资源不直售 | 销售 |
| **订单拆单** | 一订单按PhysicalItem逐件拆N条独立DeviceShipment | 销售+教务 |
| **下单即锁门店** | 订单归属锁定门店，消课/分佣/结算按此链流转 | 销售+教务 |
| **代理不参与教务** | 排课一律在总部教学部 | 教务 |
| **教师归总部** | 教师统一由总部招募管理，线上直播授课覆盖全国 | 教研+教务 |
| **销售→内容→教务联动** | CoursePackage.mapItems引用地图 → 开班排课向导 → Schedule.mapId/mapStageId/sessionSeq | 教研+销售+教务 |
| **6维能力固定不变** | 吃子/逃跑/做眼/对杀/布局/收官，教研定义后全局使用 | 教研+教务 |
| **直播课题三端统一** | 家长端/学员App/教师Web共用同一个直播课题名字 | 教研+教务 |
| **学习报告统一数据源** | 三端共享同API同字段 | 教研+教务 |
